# 2023 中国宏观货币流动可视化

## 项目主题

本项目是“数据可视化”课程作业，主题为“货币流动可视化”。项目基于 `data/macro_circulation_graph_2023.json` 中整理好的 2023 年中国宏观资金流量数据，展示住户、企业、政府、金融机构、国外部门和生产过程之间的货币循环关系。

页面包含两部分内容：

- **主页主图**：使用节点-边图展示宏观部门之间的有向加权流动，边的粗细表示流量规模。
- **主页下方统计图**：从同一份 JSON 数据中动态派生邻接矩阵、净流入/净流出、Pareto 分布、类别-部门矩阵、雷达图、类别总量和 Top 流量图。所有统计图直接排在主图下方，通过向下滚动查看。

## 技术路线与框架

项目采用原生 HTML、CSS、JavaScript 和 D3.js 实现，没有引入 npm、构建工具或后端服务。

- **D3.js**：负责读取 JSON、数据聚合、SVG 绘图、比例尺、坐标轴、缩放、拖拽和交互提示。
- **SVG**：用于绘制节点、曲线边、矩阵、柱状图、折线图和雷达图。
- **JSON 数据驱动**：主图和统计图都从 `macro_circulation_graph_2023.json` 动态生成，避免手动写死数值。
- **加权有向图分析**：使用 source、target 和 display_value 构造流量边、邻接矩阵、节点净流量和类别-部门关联矩阵。

## 遇到的困难

1. **宏观资金流量不是普通交易流水，数据解释需要谨慎**  
   JSON 中的边来自资金流量表和教学化整理，并不表示真实世界中一笔笔资金从某个部门直接转账到另一个部门。例如“居民消费支出”在会计表中更接近用途项，而在图中被简化为“住户部门 -> 企业”。因此可视化时不能只追求“路径真实”，还要在说明中强调这是部门层面的宏观账户流动。项目中通过 `plain_english`、`caveat`、类别标签和右侧解释面板降低误读风险。

2. **流量规模差异大，直接线性映射会破坏可读性**  
   数据中既有数十万亿元级别的大流量，也有几千亿元级别的小流量。如果直接使用线性比例映射边宽，大边会遮挡节点和其他边，小边则几乎不可见。项目最终使用平方根比例尺压缩极端差异，让大流量仍然突出，同时保留中小流量的视觉存在感。统计图中也统一使用“万亿元”简写，避免数字过长影响阅读。

3. **节点数量少但关系密集，边与边、边与节点容易重叠**  
   宏观部门只有几个，但同一对部门之间可能同时存在工资、税收、贷款、存款、债券等多种关系，而且方向可能相反。单纯使用力导向图会导致布局不稳定，也难以表达教学叙事。项目采用固定节点布局，再为主要边手动设置贝塞尔曲线控制点，并用边的浅色 halo、透明度和默认可见边筛选来控制视觉拥挤。后续还加入了缩放和平移功能，方便查看局部。

4. **节点拖拽与曲线边更新需要保持同步**  
   主图支持节点拖拽后，不能只移动节点本身，否则边会停留在旧位置。实现时需要在拖拽过程中实时更新节点坐标，并重新计算所有相关边的起点、终点和曲线路径。这里的难点是既要保留原本手调曲线的整体方向，又要让曲线端点跟随节点变化，所以边路径被封装为统一的 `edgePath()` 计算函数，拖拽时统一刷新。


## 运行方式

由于页面需要通过 D3 读取本地 JSON 文件，建议使用本地静态服务器打开：

```bash
python3 -m http.server 8010
```

然后在浏览器访问：

```text
http://localhost:8010/
```

如果 8010 端口被占用，可以换成其他端口，例如：

```bash
python3 -m http.server 8020
```

再访问对应端口即可。
